# 🎈 `async` vs `defer`
## 💻 `<head>`안에 script 파일을 담으면?
- 브라우저가 html을 한 줄 씩 parsing하며 Dom요소로 변환함
- `<script>`태그를 읽고 해당 경로의 js파일을 서버에서 다운받고 실행

-> 웹사이트 로딩 시간이 길어짐

## 💻 `<body>` 마지막에 script 파일을 담으면?
- 브라우저가 html파일 parsing, 페이지 준비
- 이후 `<script>` fetching

-> 사용자가 빨리 콘텐츠 볼 수 있음, 그러나 정상적인 페이지를 보는 시간 소요!

## 💻 `<head>`안에 `<script>` + `async`속성값 부여
- 브라우저가 html파일 parsing, js파일 다운 병렬 처리
- js파일 실행된 이후 나머지 html파일 parsing
- 정의된 스크립트 순서와 상관 없이 먼저 다운로드 되면 실행

->  `<body>` 마지막에 script파일 담을 때보다 시간 절약 가능,
but 정상 페이지 작동에 문제가 생김

## 💻 `<head>`안에 `<script>` + `defer`속성값 부여
- 브라우저가 html파일 parsing, js파일 다운 명령
- parsing 하는 동안 fetching
- 페이지 로드가 완료되면 순서대로 executing

# 🎈 `use strict`
- 순수 자바스크립트 사용 시 js파일 제일 처음에 쓰는 것 권장
- 유연한 언어인 javascript 사용 시 오류 방지 위해 엄격 모드를 실행하는 것

```javascript
'use strict'

console.log("Hello World!")
```
